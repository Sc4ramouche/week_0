1. Прежде чем ответить на вопросы, зачем нужна онлайн валидация и как она реализуется,
поговорим о предназначении и структуре XML-документов.

2. XML это язык разметки, определяющий правила оформления документов в формате читаемом как компьютером,
так и человеком. Принципы XML-документов это простота, всеобщность и удобство использования.
Они используются для отображения различных структур данных и передачи информации между веб-сервисами.

3. Для эффективного использовать XML-документов к ним были разработаны требования:
XML документ:
имеет один(!) корневой элемент
для каждого открывающего тэга имеет закрывающий
Тэги чувствительны к регистру
Тэги должны быть правильно вложены
Значения аттрибутов тэгов должны быть заключены в кавычки
Удовлетворяющий представленным выше требованиям документы является well-formed.
Errors in XML documents will stop your XML applications.
The W3C XML specification states that a program should stop processing an
XML document if it finds an error. The reason is that XML software should be small, fast, and compatible.
HTML browsers are allowed to display HTML documents with errors (like missing end tags).
With XML, errors are not allowed.


4. Помимо этого, документ может является валидным. Это означает, что документ имеет в себе ссылку на 
файл валидации, описывающий элементы, атрибуты и грамматические правила. XML процессоры, иначе говоря, парсеры, могут
как иметь функцию валидации, так и обходиться без неё. Процессор, способный обнаружить ошибку валидации должен
"уметь" сообщить о ней и проделать свою работу до конца.
Один из способов описания структуры XML-документа является Document Type Definition.
DTD файл составляется с использованием расширенной формы Бэкуса-Наура.
Форма Бэкуса — Наура (сокр. БНФ, Бэкуса — Наура форма) — формальная система описания синтаксиса,
в которой одни синтаксические категории последовательно определяются через другие категории.
На слайде представлен пример DTD файла, который описывает затесавшийся в уголке XML документ.

5. Также есть другой подход, который, наверное, можно назвать "схемой". Пример также представлен на слайде.
W3C рекомендует использовать Schema, так как такой формат также написан на XML, поддерживает типы данных и 
пространства имён.

6. Возвращаясь к теме моего выступления, я провёл небольшое исследование. Создав поисковый запрос, который
можно увидеть на экране, я исследовал самые ранжированные с точки зрения google инструменты онлайн-валидации.
Все три ресурса отлично справились с определением well-formed документов и все они корректно находили
ошибки. Однако я отметил, что при сознательном внесении многочисленных ошибок ресурс из третьего столбца 
добросовестно находил их все, в отличие от первых двух, которые, видимо, при обнаружении ошибки останавливали
парсинг.
Я обнаружил, что на сайте codebeautify не предусмотрена валидация ни по dtd, ни по xsm. В то время как 
xmlvalidation по заявленному функционалу производит валидацию обеими способами. Однако, предоставив
онлайн-валидатору валидный xml документ, взятый с сайта microsoft developer network, я получил
сообщение о том, что документ не валидный. Та же ситуация случилась с dtd.
Последний ресурс отметил документ как валидный, однако не поддерживает dtd валидацию.
